一年又一年 2020-08

发表于 2020-08-26 更新于 2021-06-27 分类于 随笔

个人博客，觉得最好的还是阮一峰的网络日志

从上学，到工作，写东西都不是自己的长项；可能岁数的增长，有很多东西又想写写……

不一定像《科技爱好者周刊》那样“高大上”，只想随便写写……

去年的时候，将手机改造成了服务器，同时整理了android手机上部署linux的笔记：

手机安装LinuxDeploy 使用总结 2019

手机安装Linux Deploy 构建私有云 2019-06-13

手机安装Linux Deploy 构建私有云——个人博客docsify

当时就想部署了很多服务，很多依赖的系统有些乱，以前没有使用docker，就想试试，结果开始“入坑”了……

手机上使用docker

对docker的底层技术没有了解，上来就开始在LinuxDeploy部署的centos上安装docker，运行才发向，LinuxDeploy就是创建了一个容器，里边不能再加载docker……

然后就开始查有没有办法直接在手机上运行docker，找到一个办法：

Android运行Docker！(Termux + QEMU + linux_alpine, qemu网络映射)

相当于利用QEMU在android手机上跑了一个虚拟机，搭建后发现，我的手机性能不够用……

有得有失，发现了Termux这个在android上的终端应用，可以方便快捷的在手机上使用linux，如果不习惯Termux的操作，有个神器 termux-install-linux，这里找了一篇介绍：

Android Termux 安装 Linux 就是这么简单

搭建自己的聊天服务器

家里两台电脑，分别安装了linux与windows，还有手机，有时在一台机器上上网发现些内容或顺手记录时，再到另一台电脑上整理时，不如发到IM软件上方便（至少有个开机提醒功能）。以前都是用有道笔记过度一下，linux上没有桌面程序，只能使用网页版本，感觉很不爽（心里因素：）

找了找，被riot吸引了，也被带到了坑里……

先是在欧洲的服务器matrix不能访问

然后就看到可以用docker安装，安装后发现客户端不能连接服务器，就想研究一下源码，还好github有

经历了艰苦的下载（尝试了很多增加github下载网速的方法），将源码下载，结果一编译，悲剧了，

现在不记得是哪个模块了，文档里写着，因为美国……，然后只能在欧洲的德国，荷兰……地方才能下载……

源码不全，也没过多去看，最大的收获是，发现了gitea这个git服务端管理工具，网上经常有人提到，加速github下载的方式是去gitee上做镜像，gitea也提供了类似功能，下载速度和git clone比，那就是光速了……

看了一下gitea源码，里边利用了 go-github 模块

go-github is a Go client library for accessing the GitHub API v3.

也不是每次用gitea镜像都成功，可以多尝试几次，特别是在后半夜，早晨……

相对于gitee上的镜像，gitea可以镜像发行的二进制文件

搭建下载服务器

最初买的华为路由器，里边有后台下载功能，比较“鸡肋”，只能在手机端操作……

利用aira2搭建下载服务器就比较轻松了，网上很多介绍，不多介绍，这个有两个参考：

aria2-rpc-conf 安装配置脚本

Aria2 for Chrome

android内核源码编译

手机上的linux服务器，搭建了gitea，aria2等下载功能后，觉得手机存储空间有些不够用，便想着把下载的文件直接通过网络共享存储到路由器外接的移动硬盘上，连接cifs网络共享时，发现手机的android内核没有编译，加载cifs.ko模块，也就不能访问cifs/smb网络共享，我开始进入了编译android内核的“坑”……

开始从清华，中科大镜像下载android源码，说是速度快，相对andrid源码的上百G，通宵下载是常事……

比较悲剧的是不能访问android官方网站，靠着从网上东拼西凑找的方法一点点实验……

经历很多磨难，编译了一个可以在我的oneplus2手机上加载的cifs.ko模块，结果mount网络共享文件夹时，系统崩溃重启了，只能暂时放弃……

暗黑时刻，硬盘坏了……

这段时间，我都是使用安装了linux系统的笔记本（不熟悉linux，只能平时多接触）。一段时间记录的笔记，在某一个晚上全部消失了，计算机启动时，检测不到硬盘了……

由于前半年很多尝试，没有完整的成功经验，都记录了经过，没有发布到博客上；很长一段时间，没有将本地的仓库提交到服务器备份，……

海康hik200

硬盘坏了，关注一下数据备份，买了一个海康的NAS服务器，个人用还是足够了（虽然花的钱可以买百度网盘，但百度的VIP网速，也不适合做备份），海康这款产品的优点看官方广告，只谈谈缺陷：

  • 和华为路由器一样，下载服务只能在手机端操作（不过百度网盘可以在后台慢慢同步了）
  • 使用微软的robocopy同步目录，线程开到128个，服务器死机了……
  • PC端软件下载位置在手机客户端提到了，开始没有注意，只有windows客户端，没有linux版本
  • 硬盘必须通过手机客户端格式化，否则手机端不能访问硬盘……（PC端能看到文件）
  • 用ubuntu在共享文件夹创建的目录，在windows上看不到，但是能使用全路径访问

华为路由器上的百度网盘插件已经不能使用，不清楚是不是和美国制裁有关，毕竟百度也是美国公司……

更坏的事情，windows也需要重新安装……

用的笔记本windows系统，几年了，没有重新安装过，有一天，发现cmder启动时，提示有个系统库没有加载，网上查了一下，用sfc /scannow检查，问题来了，重启后不能再进入windows，反复检查系统，只好重新安装

windows 系统重新安装

通常我习惯将笔记本显示通过HDMI接到显示器上使用，重新安装系统时也是连接着HDMI数据线到显示器（带电），安装完系统，发现不能多屏显示了。

我用的dell笔记本，已经过保质期，几乎是龟速，从dell网站下载对应驱动，重新安装后没有效果；笔记本是显卡是intel集成的，去intel网站下载驱动也不行……

不要使用dell的客户端工具下载驱动，网速不稳定，可以直接用浏览器打包下载。

再次重新安装windows，这次无意中把HDMI数据线拔了下来，安装后再次插上HDMI数据线，外接显示器显示正常了：）

win10自带驱动完全满足硬件驱动安装

刚刚结束安装进入系统时，我查看显卡驱动还是通用驱动程序，过一会就被更新为对应显示芯片驱动了（不清楚是不是win10自动联网获取了驱动，没有深究……）

Microsoft Edge

现在windows给我最大的惊喜就浏览器改用Chromium 内核了，提供了一个“集锦”功能（以前的浏览器收藏夹可以实现类似功能），操作比较直观，并且在里边可以填写注释……

浏览器同步功能，可以保存你登陆网站的用户名密码，对于我是比较方便了；敏感的站点可以选择不保存（不同步）

扩展直接使用chrome扩展

windows nodejs

nodejs安装时，多了一个选择，

安装时由于“……”原因，总是失败，后来发现主要是需要微软的c/c++编译环境 windows-build-tools ，可以通过npm手动安装：

npm install --global windows-build-tools

直接使用网站上提供指令安装失败，修改为

npm install --global windows-build-tools --vs2019

实际安装的vs2017

国内下载镜像

我用了typora markdown 文本编辑，其它的都放弃了

以前没有注意，typora提供了图片上传功能，里边使用了开源的PicGo

在文件-偏好设置-图像 里选择即可

几个typora技巧：

  • 直接粘贴剪贴板里的图片
  • 从html网页上复制表格，粘贴到typora里
  • ctrl+k 添加超级链接，光标前有文字，没有文字，剪贴板里的数据格式是超级链接，会得到不同的结果
  • 各类快捷键看菜单，为了便于记忆，有的快捷键，我在vscode中修改成一致的
